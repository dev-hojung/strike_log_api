# Strike Log API Project Guide

이 문서는 프로젝트 개발 시 준수해야 할 규칙과 지침을 담고 있습니다.

## 1. 테스트 코드 작성 규칙
- 모든 Controller 및 Service는 유닛 테스트(.spec.ts)를 포함해야 합니다.
- 테스트 파일은 대상 파일과 동일한 위치에 생성합니다.
- Mocking을 적극 활용하여 독립적인 테스트 환경을 구축합니다.

## 2. API 개발 규칙
- 새로운 엔드포인트 추가 시 반드시 테스트 코드를 동반합니다.
- 비밀번호 등 민감한 정보는 `bcrypt`를 사용하여 해싱하여 저장합니다.
- 엔티티 정의 시 `select: false`를 활용하여 민감 정보 노출을 방지합니다.

## 3. 코드 주석
- 모든 클래스와 메서드에는 `jsdoc` 형식의 상세한 주석을 작성합니다.
- 주석에는 '왜' 이 로직이 필요한지에 대한 설명을 포함합니다.
- 수정일자는 작성하지 않습니다.
